# Ghostty Cross-Platform Config Repo

A small, version-controlled Ghostty setup for macOS and Linux with a dark, muted developer theme and an optional coding prompt that shows the current Git branch.

This repo is intentionally conservative:

- One shared config path: `~/.config/ghostty/config.ghostty`
- One shared base config for both macOS and Linux
- Platform overlays generated by the installer
- One dark custom theme: `muted-ink`
- Optional machine-local overrides kept out of Git
- Optional bash/zsh prompt snippets for Git-aware coding sessions
- Optional Starship prompt config for richer language/toolchain status
- Setup docs for macOS, Linux, shell integration, fonts, themes, and troubleshooting

Ghostty already has strong defaults, including an embedded JetBrains Mono font and built-in Nerd Font support. This repo only changes subjective workflow choices: theme, padding, shell integration, scrollback, keybinds, and per-platform window behavior.

## Quick start

```bash
git clone <your-repo-url> ghostty-config
cd ghostty-config
./scripts/install.sh
./scripts/install-prompt.sh
```

Then reload Ghostty:

- macOS: `cmd+shift+,`
- Linux: `ctrl+shift+,`

Or fully restart Ghostty.

## What the Ghostty installer does

`scripts/install.sh` installs this repo's `config/` directory to:

```text
${XDG_CONFIG_HOME:-$HOME/.config}/ghostty
```

That path works on Linux and macOS. macOS also supports the native Application Support path, but the XDG path keeps dotfiles consistent across both platforms.

The installer:

1. Detects macOS vs Linux.
2. Backs up any existing Ghostty config directory.
3. Installs `config/` by symlink by default.
4. Writes `profile.ghostty` from the detected platform profile.
5. Leaves `local.ghostty` untracked for private machine-specific overrides.

## Install options

```bash
./scripts/install.sh                 # symlink config, auto-detect platform
./scripts/install.sh --copy          # copy files instead of symlink
./scripts/install.sh --profile linux # force linux profile
./scripts/install.sh --profile macos # force macOS profile
./scripts/install.sh --profile minimal
./scripts/install.sh --dry-run
```

## Git branch prompt

Ghostty does not own the shell prompt; your shell does. This repo includes bash and zsh snippets that display:

- current directory
- current Git branch
- modified files: `*`
- staged files: `+`
- untracked files: `?`
- ahead/behind upstream: `↑N` / `↓N`
- Python virtualenv or Conda environment
- previous command exit status

Install the prompt block:

```bash
./scripts/install-prompt.sh
```

Force a shell:

```bash
./scripts/install-prompt.sh --shell zsh
./scripts/install-prompt.sh --shell bash
```

Example prompt shape:

```text
~/projects/app git:main*? py:.venv
❯
```

For a richer prompt with command duration, Node version, Docker context, and optional Kubernetes context, use `config/shell/starship.toml`. See [Coding prompt](docs/PROMPT.md).

## Validate config

After installing Ghostty, run:

```bash
make validate
```

or directly:

```bash
ghostty +validate-config --config-file ./config/config.ghostty
```

The validation command is only available when the Ghostty CLI is installed and on your `PATH`.

## Customize safely

Do not edit `config/config.ghostty` for per-machine settings. Instead:

```bash
cp ~/.config/ghostty/local.example.ghostty ~/.config/ghostty/local.ghostty
$EDITOR ~/.config/ghostty/local.ghostty
```

`local.ghostty` is intentionally ignored by Git.

Common overrides:

```ghostty
font-size = 13
background-opacity = 1
background-blur = false
theme = muted-ink
```

## Recommended setup sequence

1. Install Ghostty.
2. Clone this repo.
3. Run `./scripts/install.sh`.
4. Run `./scripts/install-prompt.sh`.
5. Run `make validate`.
6. Reload or restart Ghostty.
7. Adjust `local.ghostty`, not the shared config.
